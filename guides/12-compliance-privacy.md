# 컴플라이언스 / 개인정보 처리 가이드

> 대상: 사용자 개인정보를 받는 모든 서비스 (즉, 사실상 모든 서비스).
> 한국 개인정보보호법 + EU GDPR 기본 대응. 완벽 법률 자문이 아니라 **최소한의 사고 방어**.

## 왜 이것도 보안 이슈인가

"우리 앱은 해킹만 안 당하면 돼" — 오해입니다. 컴플라이언스 위반은:
- **과징금**: 국내 개인정보보호위원회 과징금은 매출의 최대 3%. GDPR은 전 세계 매출 4% 또는 2천만 유로 중 큰 쪽.
- **브랜드 손상**: 개인정보 유출 뉴스는 보도·확산 빠름. 매출 수년 타격.
- **앱 스토어 리젝트**: Apple/Google은 개인정보 처리방침 없거나 부실하면 리뷰 반려.
- **서비스 차단**: EU는 해외 서비스에도 GDPR 적용. 위반 시 결제 차단, IP 차단.

## 피해 실예

- **개인정보 유출 후 미신고**: 법상 72시간 내 신고 의무, 미신고 시 추가 과징금
- **수집 동의 없이 마케팅 메일**: 스팸 신고 누적 → 도메인 블랙리스트, 과태료
- **탈퇴한 사용자 데이터 영구 보존**: 삭제 요청 무시하면 GDPR 위반 (Right to be forgotten)
- **미성년자 데이터 보호자 동의 없음**: 14세 미만 동의는 법정대리인 필요
- **쿠키 동의 배너 없음**: GDPR 지역 사용자에게 표시 필요, 미 표시 시 불법

## 핵심 원칙 8가지

### 1. 수집 최소화 (Data Minimization)

진짜 필요한 것만 받기:
- 회원가입에 주민번호 요구 → **위반 가능성 높음**. 본인확인이 꼭 필요하면 본인확인기관(PASS/NICE) API 사용.
- 생년월일 → 만 나이 성인 여부만 필요하면 예/아니오만.
- 전화번호 → SMS 필요 없으면 안 받기.

AI가 기본 회원가입 스키마에 이메일·비밀번호·이름·전화번호·생년월일 다 넣는 경우 많음. **정말 이 서비스에 필요한지** 질문.

### 2. 명시적 동의

```
회원가입 폼에:
[ ] 이용약관 동의 (필수)
[ ] 개인정보 수집·이용 동의 (필수)
[ ] 개인정보 제3자 제공 동의 (해당시 필수·선택 분리)
[ ] 마케팅 정보 수신 동의 (선택)

* 필수 동의 없으면 가입 불가, 선택 동의는 체크 없이도 가입 가능해야 함
* 기본값 체크 금지 (opt-in, not opt-out)
```

마케팅 메일 발송 전 **선택 동의 체크 여부 확인** 로직 필수:
```ts
if (user.marketingConsent) await sendPromo(user.email)
```

### 3. 삭제·열람·정정 요청 지원

GDPR: Right to Access / Right to Rectification / Right to Erasure / Right to Data Portability

국내법: 개인정보 열람·정정·삭제·처리정지 요구권

최소 구현:
```
설정 페이지에:
- "내 정보 다운로드" → JSON으로 저장된 모든 개인정보 export
- "내 정보 수정" → 이름·이메일 등 변경
- "회원 탈퇴" → 즉시 또는 유예 후 실제 삭제 (soft-delete가 아닌 physical delete)
- "전체 삭제 요청" 이메일 창구 명시 (법정 요구)
```

**soft-delete**(`deletedAt` 컬럼만 찍고 실제 데이터 남김)는 컴플라이언스 관점에서 불완전. 실제 삭제가 원칙이고, 유예는 30~90일 이하.

### 4. 데이터 보존 기간 명시

각 개인정보 유형별로 **보존 기간 정책** 정의하고 그 이후 자동 파기.

```
접속 기록: 3개월
주문 내역: 5년 (전자상거래법)
세금계산서: 5년
결제 정보: 수집 안 함 (PG가 보관)
마케팅 동의 철회 기록: 5년
회원 기본 정보: 탈퇴 시 즉시 삭제 (법정 보존 대상 제외)
```

Cron으로 자동 파기. 이 정책은 개인정보처리방침에 공지.

### 5. 암호화·가명화

**저장 시 암호화 필수 정보**(국내법):
- 비밀번호 (bcrypt/argon2 일방향)
- 주민번호 (받으면 안 되지만 받아야 하면 AES-256)
- 카드번호 (원칙적으로 저장 금지, PG에 위임)
- 여권번호·운전면허번호

**전송 시**: HTTPS 강제.

**가명화**: 통계·분석용으로 사용할 땐 해시·토큰화로 본인 식별 불가하게.

### 6. 유출 대응 플레이북 (미리)

사고 발생 시 시간 싸움. 미리 준비:

```
발생 인지 → 1시간 내: 담당자 소집, 범위 파악 시작
→ 24시간 내: 당국 초기 신고 준비 (정확한 규모 몰라도)
→ 72시간 내:
  - 개인정보보호위원회 + KISA 신고 (국내법 72시간)
  - GDPR 해당 시 데이터 보호 감독기관 신고 (EU 72시간)
  - 정보주체 통지 (이메일·사이트 공지)
→ 사후: 재발 방지 대책, 원인 분석 공개
```

[13-incident-response.md](./13-incident-response.md) 참조.

### 7. 개인정보처리방침 (Privacy Policy)

**필수 항목**:
1. 수집 항목 (이메일, 이름, ...)
2. 수집 목적 (회원 관리, 서비스 제공, ...)
3. 보유 기간 (탈퇴 시까지, 5년 등)
4. 제3자 제공 (해당시)
5. 처리 위탁 (AWS, Sendgrid 등)
6. 이용자 권리 (열람·정정·삭제·처리정지)
7. 쿠키 정책
8. 개인정보 보호책임자 연락처
9. 변경 이력

Apple/Google 스토어 등록 시 URL 필수. 빈 페이지 두지 말고 실제 내용 채우기.

### 8. 쿠키 동의 배너 (GDPR 지역)

```tsx
// 최소 구현
if (userInEU && !hasConsentCookie) {
  <ConsentBanner>
    필수 쿠키만 / 분석 쿠키 허용 / 마케팅 쿠키 허용
    각각 체크박스 → "동의" 버튼
  </ConsentBanner>
}

// 동의 전에는 Google Analytics / Facebook Pixel 등 로드하지 않음
```

`consent-manager`, `cookieconsent` 같은 라이브러리 활용.

## AI에게 시킬 때 덧붙일 프롬프트

````
개인정보·컴플라이언스 관련 코드 작성 시 다음 규칙:

1. 회원가입 스키마에 꼭 필요한 필드만. 주민번호·자세한 주소·생년월일 등은
   서비스 목적에 명확한 사유가 있을 때만.
2. 수집 동의는 명시적 opt-in(기본값 체크 해제). 필수/선택 분리.
3. 마케팅 메일·SMS 발송 전 해당 사용자의 marketingConsent 플래그 확인.
4. 탈퇴 로직은 soft-delete가 아닌 실제 삭제. 법정 보존 대상(주문·세금 등)만
   별도 테이블로 분리해 익명화 상태로 보존.
5. 비밀번호 bcrypt/argon2, 민감 개인정보(주민번호·여권 등 받는다면) AES-256
   암호화 저장.
6. 설정 페이지에 "내 정보 다운로드", "내 정보 수정", "탈퇴" 제공.
7. 개인정보처리방침 페이지는 9개 필수 항목 포함.
8. EU 지역 사용자에게 쿠키 동의 배너. 동의 전에 GA/Pixel 로드 금지.
9. 보존 기간 지난 데이터 자동 파기 cron 설정.
10. 유출 탐지 시 72시간 내 당국 신고 프로세스 문서화.
````

## 지뢰 10개

1. **주민번호 수집 + 평문 저장** — 국내법 심각한 위반
2. **마케팅 동의 기본 체크** — opt-out 구조는 위반
3. **탈퇴해도 DB에 남김** — Right to Erasure 위반
4. **개인정보처리방침 빈 페이지 / placeholder** — 앱 스토어 리젝트
5. **무료 분석 툴(GA/Hotjar) 동의 없이 로드** — GDPR 지역에서 불법
6. **유출 후 내부 쉬쉬** — 미신고 시 과징금 가중
7. **테스트 DB에 실제 회원 데이터 복제** — 개발자 노트북 분실 시 유출
8. **로그에 이메일·이름·주소 전체** — 로그 유출 시 개인정보 유출
9. **비밀번호 "기억하기" 토큰을 DB에 평문** — 해시 없이 영구 유효
10. **자녀 계정(14세 미만)에 보호자 동의 없음** — 법정대리인 동의 필수

## 머지 전 체크리스트

- [ ] 수집 필드 최소화 검토
- [ ] 수집·이용 동의 UI에 필수/선택 분리
- [ ] 마케팅 동의 플래그 기반 발송 로직
- [ ] 탈퇴 시 실제 삭제 (또는 법정 보존 대상만 익명화 상태로 분리)
- [ ] 데이터 다운로드 / 수정 / 삭제 API 제공
- [ ] 개인정보처리방침 9개 항목 완비
- [ ] 쿠키 동의 배너 (EU 대상이면)
- [ ] 보존 기간 정책 문서화 + 자동 파기 cron
- [ ] 민감 개인정보 암호화 저장
- [ ] 로그 PII 마스킹
- [ ] 테스트 환경에 실제 회원 데이터 없음
- [ ] 개인정보 보호책임자 연락처 공개
- [ ] 유출 대응 플레이북 문서화
